# PandaTools

A Windows system tray utility for Digital Services colleagues at Leeds Beckett University.
PandaTools provides quick access to web tools, network utilities, and PowerShell scripts fetched live from GitLab.

## Features

- System tray icon with right-click context menu
- Quick launch shortcuts for web tools and network utilities
- Live PowerShell script execution fetched directly from GitLab — always runs the latest version
- Smart script caching — scripts are cached locally and only re-downloaded when the GitLab commit hash changes
- Automatic fallback to cached scripts when GitLab is unreachable (off-network or no VPN)
- Automatic update checker on startup — prompts when a newer release is available on GitLab
- AES-encrypted GitLab API token via CredEncrypt-Utility — no plaintext credentials on disk or in memory beyond the active session

## Requirements

- Windows 11
- Access to `gitlab.leedsbeckett.ac.uk` (on-network or VPN)
- Token files generated by **CredEncrypt-Utility** placed at:
  - `C:\Windows\Build\PandaTools\K_PandaTools.txt`
  - `C:\Windows\Build\PandaTools\C_PandaTools.txt`

> PandaTools ships as a self-contained executable — no .NET runtime installation required.

## Getting Started

### 1. Generate token files

Before first run, generate your encrypted token files using CredEncrypt-Utility.
Without them, GitLab features (script runner, update checker) will be silently disabled — the tray icon will still load.

### 2. Install

1. Download `PandaToolsSetup-vX.X.X.exe` from the [Releases](../../releases) page
2. Run the installer — it extracts and launches PandaTools automatically
3. PandaTools will appear in the system tray

### 3. Usage

Right-click the tray icon to access the menu. Scripts are fetched from GitLab on first run and cached at:

```
%LOCALAPPDATA%\PandaTools\ScriptCache\
```

If a script has been updated on GitLab since your last run, PandaTools will notify you and download the latest version automatically before executing.

## Updates

PandaTools checks for a newer release on GitLab silently at startup. If an update is available, a prompt will appear with the option to open the releases page.
You can also trigger a manual check from the tray menu at any time.

## Building

Releases are built and published automatically via GitLab CI when a version tag (e.g. `v1.0.0`) is pushed.

To build manually:

```powershell
# Build main app
dotnet publish PandaTools.csproj `
  --configuration Release `
  --runtime win-x64 `
  --self-contained true `
  -p:PublishSingleFile=true `
  --output ./payload_staging

# Package payload
Compress-Archive -Path "./payload_staging/*" `
  -DestinationPath "./PandaTools.Setup/Payload/payload.zip" -Force

# Build installer
dotnet publish PandaTools.Setup/PandaTools.Setup.csproj `
  --configuration Release `
  --runtime win-x64 `
  --self-contained true `
  -p:PublishSingleFile=true `
  --output ./installer_out
```

Output: `./installer_out/PandaToolsSetup.exe`
